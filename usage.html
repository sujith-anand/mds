Color Usage

Hierarchy

When designing an application, it's crucial to establish a clear hierarchy of colors. This ensures that there is a consistent and cohesive visual language throughout the application. To establish hierachy visually in the MDS use the lighter shades (color values less than or equal to 200) to display the hierarchy of colors. Below show how you can establish hierarchy

Color Meaning

Color can communicate the meaning of different UI elements. Blue indicates information Green indicates positive action, yellow indicates a warning, and red indicates negative action. However, these could be perceived differently in different situations. It's important to consider the context in which colors are used. Here is an interesting example, if you are building a dashboard to indicate spend management and if the value of your spend increases, it's not positive and should be indicated with red. But on the other hand, if your profit increases, it is a positive situation and has to be indicated in green. Understanding the context is very important before using any color.

State

Color can provide information about the state of an app, its components, and elements. This includes: Current state of an element or component, such as whether a button is enabled or disabled

  • Changes in state to an app, component, or element.
  • Color should be noticeable when indicating state changes, as subtle differences in color may be missed
